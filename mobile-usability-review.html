Kernel Panic — Mobile Usability Review

390×844 viewport (iPhone 14) · March 28, 2026 · Code + visual audit

1. Hero Slide

#hero — First impression

High

Navigation buttons completely hidden

The .buttonBar with "BUY NOW", "Back", and "Next" buttons is cut off below the viewport fold on every slide. The button bar competes for space with the full-bleed hero image. On a 390×844 phone, the Win95 window chrome (title bar + bevels) eats ~70px, and the button bar needs ~45px, but the content area fills 100dvh with no guarantee the buttons remain visible. Users have no visible way to navigate forward.

Medium

No swipe gesture navigation

Mobile users instinctively swipe to navigate slides. The current implementation only supports button taps (which are hidden) and click-to-advance (tapping the right 75% of the content area). There's no touchstart/touchend swipe handler. For a slide-based experience, swipe is expected and its absence makes the site feel broken.

Medium

No visual affordance for tap-to-navigate

The hidden click zones (left 25% = back, right 75% = forward) have zero visual indication. Desktop users get a custom cursor that changes color to hint at navigation, but mobile hides the cursor entirely. A first-time visitor has no way to discover they can tap to advance.

Positive

Image fills the viewport beautifully

The hero image with object-fit: cover makes strong use of the full screen real estate. The card spread photograph is compelling at mobile scale.

2. Text Slide

#text — Game description + logo

High

Wall of text with no visual breaks

Three dense paragraphs of body copy render as an unbroken block. On a 390px screen at clamp(13px, 3.5vw, 16px), this creates a long scroll with no breathing room. Mobile users scan, not read — this slide needs chunking, pull-quotes, or progressive disclosure to hold attention.

Medium

Logo dominates above-the-fold space

The KERNEL PANIC logo takes up roughly the top third of the viewport before any description text appears. Consider reducing the mobile logo size further (currently clamp(32px, 10vw, 60px) / clamp(48px, 15vw, 90px)) or hiding it on this slide since users already saw it in the title bar.

Low

Scrollbar hidden but scroll is available

The text is vertically scrollable (overflow-y: scroll with scrollbar-width: none), but there's no visual hint that more content exists below the fold. A fade gradient at the bottom or scroll indicator would help discoverability.

Positive

Typography and contrast are solid

The Redaction display font renders clearly on mobile. White heading on black with gray body text provides good hierarchy. Line-height at 1.5 is comfortable for mobile reading.

3. Split Left Slide

#splitL — Hand grabbing card

Positive

Smart decision to split the two images

Breaking the desktop's side-by-side split into two separate mobile slides is the right call. Each image gets the full viewport width, making the product photography much more impactful than a cramped 50/50 split would be.

Low

Image letterboxing

The image uses object-fit: cover which fills the frame well, but the aspect ratio difference between the source photo and the mobile viewport means some cropping occurs at top/bottom. The card in the hand is still clearly visible, so this is acceptable.

4. Split Right Slide

#splitR — Hand with clickers

Positive

Strong visual continuity

The matching blue sky/green grass background creates a cohesive feel across both split slides. The colorful clickers are eye-catching at mobile scale.

Low

Two image-only slides in a row feel thin

Without any text or context, these two slides feel like filler. Consider adding a brief caption overlay (e.g., "Play action cards in real time" / "Track scores with clicker tokens") to give each image purpose.

5. Cards Carousel Slide

#cards — Card types showcase

High

Cards not properly centered — first card offset

The mobile carousel renders with the first card (Tasks) pushed to center-left and the second card's edge peeking in from the right. The scroll-snap-type: x mandatory with scroll-snap-align: center should center cards, but the initial scroll position isn't set. The first card should be centered on load. Add padding-left equal to half the viewport minus half the card width to achieve proper centering of the first item.

High

Massive empty teal space above the card

The card sits in the lower third of the content area with a large blank teal region above it. This happens because the carousel uses align-items: center but the card's max-width: 200px combined with flex: 0 0 70% creates a small card that doesn't fill the available height. The vertical centering makes it look lost. Consider increasing the card size or top-aligning the layout on mobile.

Medium

No horizontal scroll indicator

The carousel scrollbar is hidden (scrollbar-width: none) and there are no pagination dots or other affordance showing that more cards exist to the right. Users may not realize they can swipe through four different card types. Add dot indicators or a "swipe for more" hint.

Medium

Card flip animation stagger may not work well

The desktop flip stagger (animation-delay nth-child selectors) targets .carouselItem, but mobile cards use .carouselItemMobile which doesn't have matching stagger CSS. All mobile cards may flip simultaneously instead of one-by-one, losing the polished effect.

Low

Description text clips at carousel edge

The "Actions" card's description text ("Actions are operatio...") gets clipped at the right viewport edge. This is expected during scroll, but on initial load it creates a janky impression.

6. Buy Slide

#buy — Purchase CTA

High

Price is completely hidden on mobile

The CSS rule .buySlidePrice { display: none } inside the mobile media query hides the $35 price tag. This is a critical omission — users see a "BUY NOW" button with no price context. This will significantly hurt conversion. The price should always be visible, especially on mobile where the buy decision happens fast.

High

BUY NOW button cut off at bottom edge

The button appears partially below the viewport fold. The stacked layout (image on top, CTA pane on bottom with flex: 0 0 auto) pushes the button down when the product image takes most of the vertical space. The button needs to be fully visible without scrolling — consider reducing the image height or overlaying the CTA.

Medium

Logo and subtext also hidden

.buySlideLogoContainer and .buySlideSubtext are set to display: none on mobile. This means users only see the product image and a bare "BUY NOW" button with no branding context. The "Opens Square checkout" subtext is especially important for trust — users want to know where the button takes them.

Medium

Idle popup easter egg may overwhelm mobile

The idle popup spawner at scale(0.4) on mobile creates miniature Win95 windows over the content. On a small screen these can obscure the BUY NOW button and product image. The popups don't have a close button and accumulate every second. Consider disabling or limiting this feature on mobile.

Positive

Product image looks great

The open box shot fills the top portion of the viewport nicely with object-fit: cover. The turquoise box interior with visible card decks and clickers is compelling.

7. Cross-Slide / Global Issues

Issues affecting the entire mobile experience

High

No swipe navigation anywhere

The entire slide deck relies on button presses to navigate, but on mobile the buttons are frequently obscured. Swipe left/right to navigate slides is the standard expectation for this type of content on mobile. Implement touchstart/touchend delta detection with a 50px threshold to trigger goNext()/goBack().

High

Button bar visibility is unreliable

The button bar uses a fixed amount of vertical space at the bottom of the window body, but the content area above it often pushes it partially or fully off-screen. The flex-shrink: 0 on the button bar should prevent it from being compressed, but the overall 100dvh viewport constraint combined with the title bar, bevels, and content well doesn't leave enough room. Consider making the button bar position: sticky at the bottom or switching to a fixed bottom bar on mobile.

Medium

No mobile-specific loading state

The Cloudinary images are large (the hero spread photo). On slower mobile connections, users may see a black rectangle for several seconds before images load. Consider adding loading="eager" for the hero and loading="lazy" for later slides, plus placeholder blur-up or a loading shimmer in the content area.

Medium

Win95 chrome pixel size increases on mobile

The --px variable increases from 2.5px to 3px on mobile (globals.css line 94-98). This makes the window bevels, title bar, and button bar consume more proportional space on an already tight mobile viewport. On a 390px-wide screen, the outer bevels alone eat ~12px total width. Consider keeping --px at 2px or even smaller on mobile to maximize content area.

Medium

Title bar click-to-home not discoverable

Clicking the title bar navigates to slide 0 (onClick={() => changeSlide(0)}), which is a useful shortcut but completely hidden. The cursor: pointer hint is lost on touch devices. Consider adding a small home icon or making the title bar tap behavior more obvious.

Low

Keyboard navigation not relevant but still active

Arrow key handlers are bound globally. While harmless on mobile, the keyboard listener adds unnecessary overhead. Gate with if (isMobile) return; in the keyboard handler effect.

Low

Mouse velocity logo flicker runs on mobile

The requestAnimationFrame loop that tracks mouse velocity and updates logoFont runs continuously on mobile even though there's no mouse. The velocity will always be 0 and the font won't change, but the RAF loop still ticks. Gate with if (isMobile) return;.

Priority Action Items

Ranked by impact on mobile conversion and user experience:

  1. Add swipe navigation. Implement touch swipe left/right across all slides. This is the single biggest gap — without it, many mobile users will get stuck on the hero slide.
  2. Fix the button bar visibility. Make navigation buttons always visible on mobile. Either use a sticky/fixed bottom bar or reduce content height to guarantee the buttons fit within 100dvh.
  3. Show the price on the Buy slide. Un-hide .buySlidePrice on mobile and ensure "$35" is visible above the BUY NOW button. Hidden pricing kills conversion.
  4. Fix the Cards carousel centering. Add initial padding so the first card is centered, increase card size to fill more of the teal area, and add pagination dots.
  5. Break up the Text slide wall of text. Add visual breaks, icons, or collapse the description into key bullet points with a "read more" expansion.
  6. Add captions to the Split image slides. Overlay brief descriptive text so these slides communicate product value, not just show pretty photos.
  7. Reduce Win95 chrome size on mobile. Decrease --px to 2px on mobile to reclaim viewport space for content.
  8. Gate desktop-only effects. Skip the mouse velocity RAF loop, keyboard listeners, and idle popup spawner on mobile for performance.
